## プロジェクト概要

ぬいぐるみをバックパックとして背負い、腕をサーボで動かすロボットを製作中。
商品化・販売を目指している。将来的に会話機能（音声認識→LLM→TTS）も追加予定。

---

## ハードウェア確定情報

| 項目 | 内容 |
|---|---|
| マイコン | ESP32-S3（※S3であることに注意。ESP32と間違えないこと） |
| サーボ左腕 | SG90 / GPIO14 |
| サーボ右腕 | SG90 / GPIO15 |
| ボタンA | GPIO12（短押し=左腕、長押し1秒=自動モードON/OFF） |
| ボタンB | GPIO13（短押し=右腕） |
| LED | GPIO2（自動モードON時点灯） |
| COMポート | COM5 |
| 給電 | USB（テスト中）→ 後で18650×2本化予定 |

## サーボPWM値（データシート実測値）

- 周期: 20ms（50Hz）
- 中立 (0°):  1.45ms → duty_u16 = 4751
- +90°(右端): 2.40ms → duty_u16 = 7864
- -90°(左端): 0.50ms → duty_u16 = 1638

## main.pyの現在の実装内容

### アーキテクチャ
asyncioで以下を並列実行：
- wave_runner（常駐タスク）: asyncio.Eventを監視してサーボを動かす
- button_task: 物理ボタンを5msポーリング
- auto_task: 自動モード時に左右交互に動かす
- handle_client（WebサーバーのHTTPハンドラ）

### サーボ動作シーケンス（1往復）
CENTER(400ms) → UP(500ms) → CENTER(400ms) → duty_u16(0) でPWM停止

### WebUI
- WiFi APモード: SSID=PuppetBot, IP=192.168.4.1
- スマホブラウザからアクセスして操作
- エンドポイント: GET /, GET /status, POST /wave/left, /wave/right, /wave/both, /auto, /speed

### デバウンス方式
ステートマシン（idle→debouncing→pressed）で50ms安定してから確定。
リリース時（ボタンを離したとき）に動作発火。

## これまでに対処した問題と解決策

### 問題1：チップ誤認
- 状況: ESP32と思っていたがESP32-S3だった
- 対処: esptoolで --chip esp32s3 を指定、ファームウェアもS3用を使用

### 問題2：サーボが小刻みに動く（初期）
- 原因: repeats=3（3往復）設定だった
- 対処: repeats=1（1往復）に変更、sleep_msも300→500msに延長

### 問題3：サーボが唸る（動作後）
- 原因: PWM信号を流し続けると位置保持しようとして唸る
- 対処: 動作完了後に servo.duty_u16(0) でPWM停止

### 問題4：一回のクリックで複数回動く
- 原因①: create_task()とis_wavingフラグセットの間にレース条件があった
- 原因②: ボタンのチャタリング
- 対処①: タスク生成前にis_waving=Trueをセット
- 対処②: ステートマシン方式デバウンス実装

### 問題5：4回目以降に動作がばらつく
- 原因: create_task()のたびにコルーチンオブジェクトが生成され、GCが走って
         タイミングがずれる
- 対処: 常駐wave_runner＋asyncio.Event方式に変更（メモリ確保ゼロ）
         動作後にgc.collect()を明示呼び出し

## 次フェーズの予定（まだ未着手）

### フェーズ3：電源・フレーム
- フレーム: 3Dプリント（PLA）/ 幅150×高さ200×奥行40mm / 3パーツ分割
- 電源: 18650×2本 + DC-DCコンバータ（5V出力）

- フェルト帯 + 釣りスナップ(#12) / マグネットで位置決め

- 構成: USBマイク → Orange Pi Zero 2W（Whisper STT）→ Claude API → TTS → スピーカー
- Orange Pi → ESP32 は UART接続
- コマンド: L（左腕）/ R（右腕）/ B（両腕）の1文字
- ESP32側にUART受信処理を追加する必要あり
